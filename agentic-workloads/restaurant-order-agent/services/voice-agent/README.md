# Voice Agent — Pipecat + Deepgram on AgentCore

Real-time voice pipeline built with the Pipecat framework, Deepgram STT/TTS, and
Claude Sonnet (via Amazon Bedrock), deployed to AWS Bedrock AgentCore Runtime.

```
Client audio → WebSocket (/ws) → Deepgram STT (Nova-2) → Claude Sonnet (Bedrock)
            → Deepgram TTS (Aura-2) → WebSocket → Client
```

- **STT**: Deepgram Nova-2 — low-latency speech recognition
- **TTS**: Deepgram Aura-2 Andromeda — natural voice synthesis
- **VAD**: Silero — voice activity detection for natural turn-taking
- **LLM**: Claude Sonnet via Amazon Bedrock
- **Transport**: WebSocket (bidirectional audio streaming)
- **Runtime**: AWS Bedrock AgentCore with persistent memory (STM + LTM)

## Prerequisites

- Python 3.12
- AWS CLI configured with valid credentials
- `bedrock-agentcore-starter-toolkit` (`pip install bedrock-agentcore-starter-toolkit`)

## Setup

```bash
cd services/voice-agent
python3.12 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

cp .env.example .env   # fill in DEEPGRAM_API_KEY, AGENTCORE_MEMORY_ID, etc.
```

## Local development

```bash
agentcore dev
agentcore invoke --dev '{"prompt": "Hello, how are you?"}'
```

## Deploy to AgentCore

```bash
agentcore configure \
  --entrypoint src/main.py \
  --name voiceagent \
  --non-interactive \
  --deployment-type direct_code_deploy \
  --runtime PYTHON_3_12

agentcore deploy
```

> **Note on `.bedrock_agentcore.yaml`:** This file is generated by `agentcore
> configure` and contains your AWS account ID, region, and resource ARNs. It is
> intentionally **gitignored** — run `agentcore configure` to generate it for
> your own account. This service defaults to the `us-west-2` region; pass
> `--region <your-region>` to `agentcore configure` (and set `AWS_REGION` in
> `.env`) to deploy elsewhere.

## Environment Variables

| Variable | Required | Description |
|----------|----------|-------------|
| `DEEPGRAM_API_KEY` | Yes | Deepgram API key for STT and TTS |
| `AWS_REGION` | Yes | AWS region for Bedrock and AgentCore |
| `AGENTCORE_MEMORY_ID` | No | AgentCore Memory ID (memory features disabled if unset) |

See `.env.example` for the full template. Never commit your `.env`.

## How It Works

1. Client connects via WebSocket to `/ws`.
2. Audio frames stream in from the client microphone.
3. Silero VAD detects when the user starts/stops speaking.
4. Deepgram STT (Nova-2) transcribes speech to text in real time.
5. Claude Sonnet generates a response and selects restaurant tools as needed.
6. Deepgram TTS (Aura-2) synthesizes the response to speech.
7. Audio frames stream back to the client.

The pipeline supports interruptions — if the user speaks while the agent is
responding, the agent stops and listens.
