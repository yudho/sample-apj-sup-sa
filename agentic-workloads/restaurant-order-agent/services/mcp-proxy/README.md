# MCP Proxy

A lightweight MCP (Model Context Protocol) server deployed on AWS Bedrock
AgentCore Runtime. It exposes the restaurant customer tools over JSON-RPC 2.0
and forwards calls to the HTTP restaurant backend, bridging AgentCore's HTTPS
requirement to the backend API.

## Tools exposed

`request_otp`, `verify_otp`, `list_menu`, `get_menu_item`, `get_cart`,
`add_cart_item`, `place_order`, `get_current_order`, `get_order`,
`get_delivery_status`, `get_profile`, `update_profile`.

## Setup

```bash
cd services/mcp-proxy
pip install -r requirements.txt
```

Set the backend connection via environment variables (see your deployment
config):

| Variable | Description |
|----------|-------------|
| `RESTAURANT_API_BASE` | Base URL of the restaurant backend API |
| `RESTAURANT_API_KEY` | API key for the restaurant backend |

## Deploy to AgentCore

```bash
agentcore configure --entrypoint src/main.py --name mcpproxy --non-interactive
agentcore deploy
```

> **Note on `.bedrock_agentcore.yaml`:** This file is generated by `agentcore
> configure` and contains your AWS account ID, region, and resource ARNs. It is
> intentionally **gitignored** — run `agentcore configure` to generate it for
> your own account. This service defaults to the `ap-south-1` region; pass
> `--region <your-region>` to `agentcore configure` to deploy elsewhere.
