# Llama-4-Scout 17B-16E benchmark

Per-model configuration for [`meta-llama/Llama-4-Scout-17B-16E-Instruct`](https://huggingface.co/meta-llama/Llama-4-Scout-17B-16E-Instruct),
a 109B mixture-of-experts model (16 experts × 17B per expert; ~17B active
per token) under the **Llama-4 community license, gated**. This folder is
the only place Llama-4-Scout-specific facts live — everything else is
generic infrastructure in [`src/vllm_ec2_bench/`](../../src/vllm_ec2_bench/).

## Layout

```
models/llama_4_scout_17b/
├── __init__.py          # Re-exports MODEL spec, EXPERIMENTS, prompts, catalog factories
├── model_spec.py        # LLAMA_4_SCOUT_17B: ModelSpec (gated=True)
├── experiments.py       # EXPERIMENTS: 2 ExperimentConfig instances
├── prompts.py           # Travel-booking JSON-extraction SYSTEM_PROMPT + SEED_INPUT
├── catalog_cache.json   # Hardware + prices cache (schema v2, checked-in)
└── llama-4-scout-17b-vllm-ec2-benchmark.ipynb       # Generated notebook (40 cells, 2 exps)
```

## Experiments (Llama-4-Scout 17B-16E at BF16, ~218 GiB weights)

Only 2 experiments — the 109B-parameter weight footprint needs at least
8× A100-class GPUs and benefits enormously from FP8 KV cache to fit
context within VRAM. Smaller GPU classes are skipped.

| # | Instance | GPUs | TP | DP | PP | Replicas | Notes |
|---|---|---|---|---|---|---|---|
| exp_6 | `p4d.24xlarge`  | 8× A100-40GB | 8 | 1 | 1 | 1 | **requires `--kv-cache-dtype fp8`** to fit |
| exp_7 | `p4de.24xlarge` | 8× A100-80GB | 8 | 1 | 1 | 1 | larger KV headroom; FP8 still recommended |

Experiment IDs `exp_1`–`exp_5` are deliberately skipped to keep the index
aligned with the smaller-model lineup (1=g5, 2=g6, 3=g6e, 4=g7e.2x,
5=g7e.12x). Llama-4-Scout doesn't fit on any of those.

Spot first, on-demand fallback. All experiments default to `us-west-2`.

## Prerequisites

This model is gated. You need:

1. A Hugging Face account with access to
   `meta-llama/Llama-4-Scout-17B-16E-Instruct` (accept the license at
   the HF model page first).
2. `HF_TOKEN` environment variable set when launching the experiment —
   the runtime container reads it via Secrets Manager and passes it to
   vLLM as `HF_TOKEN`.

## Task

Travel booking JSON extraction. The endpoint is benchmarked against records
generated by `sample-data/scripts/synthesize.py --domain travel`.

## Usage

```python
import os
from vllm_ec2_bench import DeploymentRunner
from models.llama_4_scout_17b import EXPERIMENTS, SYSTEM_PROMPT, load_catalog

CATALOG = load_catalog(offline_ok=False)
cfg = EXPERIMENTS["exp_6"]   # or exp_7 for A100-80GB
runner = DeploymentRunner(cfg, catalog=CATALOG, hf_token=os.environ["HF_TOKEN"])
state = runner.launch()
# ... use state.base_url / state.api_key ...
runner.terminate()
```
