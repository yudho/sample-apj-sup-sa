# Qwen3-8B benchmark

Per-model configuration for [`Qwen/Qwen3-8B`](https://huggingface.co/Qwen/Qwen3-8B),
a dense 8B Apache-2.0 model (32K native context). This folder is the only
place Qwen3-8B-specific facts live - everything else is generic infrastructure
in [`src/vllm_ec2_bench/`](../../src/vllm_ec2_bench/).

## Layout

```
models/qwen3_8b/
├── __init__.py          # Re-exports MODEL spec, EXPERIMENTS, prompts, catalog factories
├── model_spec.py        # QWEN3_8B: ModelSpec
├── experiments.py       # EXPERIMENTS: 7 ExperimentConfig instances
├── prompts.py           # Travel-booking JSON-extraction SYSTEM_PROMPT + SEED_INPUT
├── catalog_cache.json   # Hardware + prices cache (schema v2, checked-in)
└── README.md
```

## Experiments (Qwen3-8B at BF16, ~17 GiB)

| # | Instance | GPUs | TP | DP | PP | Replicas | Notes |
|---|---|---|---|---|---|---|---|

| exp_4 | `g7e.2xlarge`  | 1x Blackwell   | 1 | 1 | 1 | 1 | smallest single-GPU Blackwell |
| exp_5 | `g7e.12xlarge` | 2x Blackwell   | 1 | 2 | 1 | 2 | 1 replica/GPU |
| exp_6 | `p4d.24xlarge` | 8x A100-40GB   | 1 | 8 | 1 | 8 | dense pack |
| exp_7 | `p4de.24xlarge`| 8x A100-80GB   | 1 | 8 | 1 | 8 | huge KV headroom |

Spot first, on-demand fallback, ODCR last resort.

## Task

Travel booking JSON extraction. The endpoint is benchmarked against records
generated by `sample-data/scripts/synthesize.py --domain travel`.

## Usage

```python
from vllm_ec2_bench import DeploymentRunner
from models.qwen3_8b import EXPERIMENTS, SYSTEM_PROMPT, load_catalog

CATALOG = load_catalog(offline_ok=False)
cfg = EXPERIMENTS["exp_3"]
runner = DeploymentRunner(cfg, catalog=CATALOG)
state = runner.launch()
# ... use state.base_url / state.api_key ...
runner.terminate()
```
