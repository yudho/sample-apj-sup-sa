# Gemma-4 31B benchmark

Per-model configuration for [`google/gemma-4-31B-it`](https://huggingface.co/google/gemma-4-31B-it),
a 31B-parameter Apache-2.0 instruction-tuned model (released April 2026,
ungated). This folder is the only place Gemma-4-31B-specific facts live —
everything else is generic infrastructure in
[`src/vllm_ec2_bench/`](../../src/vllm_ec2_bench/).

## Layout

```
models/gemma_4_31b/
├── __init__.py          # Re-exports MODEL spec, EXPERIMENTS, prompts, catalog factories
├── model_spec.py        # GEMMA_4_31B: ModelSpec
├── experiments.py       # EXPERIMENTS: 7 ExperimentConfig instances
├── prompts.py           # Travel-booking JSON-extraction SYSTEM_PROMPT + SEED_INPUT
├── catalog_cache.json   # Hardware + prices cache (schema v2, checked-in)
└── gemma-4-31b-vllm-ec2-benchmark.ipynb             # Generated notebook
```

## Experiments (Gemma-4 31B at BF16, ~63 GiB)

| # | Instance | GPUs | TP | DP | PP | Replicas | Notes |
|---|---|---|---|---|---|---|---|
| exp_1 | `g5.12xlarge`  | 4× A10G        | 4 | 1 | 1 | 1 | TP=4 to fit weights |
| exp_2 | `g6.12xlarge`  | 4× L4          | 4 | 1 | 1 | 1 | TP=4 to fit weights |
| exp_3 | `g6e.12xlarge` | 4× L40S        | 2 | 2 | 1 | 2 | 2 replicas, TP=2 each |
| exp_4 | `g7e.2xlarge`  | 1× Blackwell   | 1 | 1 | 1 | 1 | smallest fit |
| exp_5 | `g7e.12xlarge` | 2× Blackwell   | 1 | 2 | 1 | 2 | 1 replica per GPU |
| exp_6 | `p4d.24xlarge` | 8× A100-40GB   | 2 | 4 | 1 | 4 | 4 replicas, TP=2 each |
| exp_7 | `p4de.24xlarge`| 8× A100-80GB   | 1 | 8 | 1 | 8 | dense pack, 1/GPU |

Spot first, on-demand fallback. All experiments default to `us-west-2`.

## Task

Travel booking JSON extraction. The endpoint is benchmarked against records
generated by `sample-data/scripts/synthesize.py --domain travel`.

Gemma-4-31B is ungated (unlike most Google releases), so no HF token is
required.
